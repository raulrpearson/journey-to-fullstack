import Text from '../components/Text';
import Paragraph from '../components/Paragraph';
import '../styles/default.css';

import { graphql } from 'gatsby';
export const pageQuery = graphql`
  query {
    site {
      siteMetadata {
        title
        description
      }
    }
  }
`;

# Accessing pageQuery props

After going through the tests below, I settled on the following solution:

```jsx
// Text.jsx
import React from 'react';
export default props => <>{props.text}</>;

// Paragraph.jsx
import React from 'react';
export default props => <p>{props.text}</p>;
```

Not sure if this optimal but it produces the desired result:

```
#### <Text text={props.data.site.siteMetadata.title} />

<Paragraph text={props.data.site.siteMetadata.description} />

- <Text text={props.data.site.siteMetadata.description} />
- <Text text={props.data.site.siteMetadata.description} />

> <Paragraph text={props.data.site.siteMetadata.description} />

What about inserting property values within a given block of text? The title of
this document is "<Text text={props.data.site.siteMetadata.title} />" and the
description is _<Text text={props.data.site.siteMetadata.description} />_. You
can also include it within a `<span>` like this: <span style="color:
#daa;"><Text text={props.data.site.siteMetadata.description} /></span>.
```

## Headings

It seems that HTML in Markdown headings is ignored. The wrapping tags have
effect, but the property references are not resolved into their values.

```
### <span>{props.data.site.siteMetadata.title}</span>

### <div>{props.data.site.siteMetadata.title}</div>

### <Text text={props.data.site.siteMetadata.title}/>
```

### <span>{props.data.site.siteMetadata.title}</span>

### <div>{props.data.site.siteMetadata.title}</div>

### <Text text={props.data.site.siteMetadata.title}/>

---

## Paragraphs

The following look the same, but the resulting HTML markup is different (inspect)

```
<div>{props.data.site.siteMetadata.description}</div>

<p><Text text={props.data.site.siteMetadata.description} /></p>

<Text text={props.data.site.siteMetadata.description} />

<Paragraph text={props.data.site.siteMetadata.description} />
```

<div>{props.data.site.siteMetadata.description}</div>

<p><Text text={props.data.site.siteMetadata.description} /></p>

<Text text={props.data.site.siteMetadata.description} />

<Paragraph text={props.data.site.siteMetadata.description} />

## Lists

Here, wrapping in `<span>` works, but also adds a nesting that we wouldn't
necessarily want to create if we were writing the markup ourselves:

```
- <Text text={props.data.site.siteMetadata.description} />
- <span>{props.data.site.siteMetadata.description}</span>
```
